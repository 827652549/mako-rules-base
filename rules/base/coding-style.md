
# 编码风格规范（通用）

> 平台专属格式约定（缩进、行宽、命名）见各平台规范。

## 注释
- 代码注释用**英文**，说明文档用中文
- 复杂逻辑必须加行内注释，说明「为什么」而非「做了什么」
- TODO/FIXME 使用统一格式：`// TODO(username): 内容`
- 禁止提交只有 `TODO` 而无说明的空壳注释

## 代码质量（通用原则）
- **禁止魔法数字**：有意义的数值提取为命名常量
- **禁止死代码**：注释掉的代码块不提交，用 git 记录历史
- **类型安全**：避免 `any` / 裸 `object`，优先具体类型或 `unknown`
- **未使用变量**：真正使用或移除，不用禁用规则掩盖

## JS/TS 项目额外约定
- 2 空格缩进，行宽不超过 100 字符，文件末尾留一个空行
- 使用分号结束语句
- 命名：变量/函数 camelCase，类/组件 PascalCase，常量 UPPER_SNAKE_CASE
- lint 检查见 `lint-requirements.md`
