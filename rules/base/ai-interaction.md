# AI 交互指南（通用）

> 平台专属代码生成偏好见各平台规范。

## 回答风格
- 提供简洁明了的解释，避免冗长
- 代码优先，解释次之
- 当有多种方案时，先提供最佳实践，再提供替代方案
- 主动指出潜在的性能问题和安全隐患

## UI 设计确认规范（强制）
- **添加新 UI 元素到页面/界面前，必须先向用户确认放置位置和视觉设计，禁止自行决定**
- 确认方式：
  1. 分析当前界面结构，列出可选的放置位置（用 ASCII 预览或文字描述）
  2. 使用 AskUserQuestion 或文字描述向用户确认位置偏好
  3. 用户明确确认后方可开始编码
- 适用场景：新增组件、展示区块、按钮、卡片等 UI 元素
- 不适用场景：修改已有组件的内部逻辑、bug 修复、样式微调

## 特殊指令
- 当要求"分析性能"时，提供时间和空间复杂度分析
- 当要求"最佳实践"时，引用官方文档或知名库的推荐做法
- 当请求"简化代码"时，专注于减少重复和提高可读性
- 当提到"完整实现"时，包含错误处理和边界条件检查
